# :material-source-branch: Next Level Selection Tasks

These tasks combine the skills from:

- :material-monitor: Outputs, inputs and variables
- :material-database-outline: Data types and arithmetic
- :material-call-split: Selection with `if` and `else`

!!! note "Important information before you begin........."

    You do not need to record your results for these tasks.

    This is a test in resilience: try things, fix errors, and keep improving your code.

    > Remember, we learn through making mistakes.

---

## :material-tools: Rules for These Tasks

For every task:

- Use clear variable names.
- Ask the user for input where needed.
- Convert numbers using `int()` or `float()`.
- Use `if` and `else` to make decisions.
- Display clear messages for both possible outcomes.
- Test your program with values that make each outcome happen.

!!! warning "Important"

    These tasks use the skills from lessons 1, 2 and 3.

    You should use `if`, `else`, comparison operators and correct indentation.

## :material-format-list-checks: Challenge Tasks

!!! question "Start where you feel confident, then try the next level."

    === "Level 1"

        **Task 1: Age Checker**

        Ask the user for their age.

        If they are `12` or older, display:

        ```text
        You can join the club.
        ```

        Otherwise, display:

        ```text
        You are not old enough yet.
        ```

        **Task 2: High Score Checker**

        Ask the user for their game score.

        If the score is greater than `1000`, display:

        ```text
        High score achieved!
        ```

        Otherwise, display:

        ```text
        Keep practising.
        ```

    === "Level 2"

        **Task 3: Password Checker**

        Ask the user to enter a password.

        If the password is equal to `python`, display:

        ```text
        Access granted.
        ```

        Otherwise, display:

        ```text
        Access denied.
        ```

        **Task 4: Temperature Warning**

        Ask the user to enter a temperature.

        If the temperature is less than `0`, display:

        ```text
        Warning: freezing conditions.
        ```

        Otherwise, display:

        ```text
        Temperature is above freezing.
        ```

    === "Level 3"

        **Task 5: Cinema Ticket Checker**

        Ask the user for:

        - Their name
        - Their age

        If they are `15` or older, display a message saying they can watch the film.

        Otherwise, display a message saying they are too young.

        Your message should include their name.

        **Task 6: Test Result**

        Ask the user for:

        - Their name
        - Their test score
        - The maximum possible score

        Calculate their percentage.

        If the percentage is greater than or equal to `50`, display a pass message.

        Otherwise, display a try again message.

        **Formula:** `percentage = score / maximum * 100`

    === "Level 4"

        **Task 7: Discount Calculator**

        Ask the user for:

        - The price of an item
        - Their age

        If the user is under `16`, they receive a `20%` discount.

        Otherwise, they pay the full price.

        Display:

        - The original price
        - The discount amount
        - The final price

        **Hint:** `discount = price * 0.20`

        **Task 8: Delivery Charge**

        Ask the user for the total cost of their shopping basket.

        If the total is greater than or equal to `25`, delivery is free.

        Otherwise, add a delivery charge of `3.99`.

        Display:

        - Basket total
        - Delivery cost
        - Final amount to pay

    === "Challenge"

        **Task 9: Simple Login and Spend Checker**

        Ask the user for:

        - Their username
        - Their password
        - The amount they want to spend

        If the password is equal to `python123`, display a welcome message using their username.

        Otherwise, display an access denied message.

        Then create a second decision:

        If the amount they want to spend is greater than `50`, display:

        ```text
        Manager approval needed.
        ```

        Otherwise, display:

        ```text
        Purchase approved.
        ```

        **Task 10: Event Ticket Program**

        Create a program for an event ticket desk.

        Ask the user for:

        - Their name
        - Their age
        - How many tickets they want

        A ticket costs `8.50`.

        Calculate the ticket total.

        If the user is `13` or older, display:

        - A message saying they can attend
        - Their name
        - Number of tickets
        - Total cost

        Otherwise, display a message saying they must attend with an adult.

        **Extra challenge:** Add a booking fee of `1.50` to the total cost.

## :material-lightbulb-on: Challenge Tips

- Write the input lines first.
- Convert numeric inputs before comparing them.
- Remember that `=` stores a value and `==` compares a value.
- Put a colon `:` after every `if` and `else`.
- Indent the code that belongs inside each decision.
- Test each task twice: once for the `if` outcome and once for the `else` outcome.
