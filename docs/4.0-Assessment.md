# :material-clipboard-text-outline: Python Programming Assessment

This assessment checks the skills you have learned in the Python basics unit.

You will use:

- :material-monitor: Outputs
- :material-keyboard: Inputs
- :material-variable: Variables
- :material-database-outline: Data types
- :material-calculator: Arithmetic
- :material-call-split: Selection with `if` and `else`

---

## :material-stairs: Assessment Levels

Each level becomes more challenging.

Start at the level your teacher tells you to complete.

| Level | Main Skills |
|-----------|-----------|
| [Level 1](4.1-level-1.md) | Read, correct and explain simple Python code. |
| [Level 2](4.2-level-2.md) | Create simple input, storage and output programs. |
| [Level 3](4.3-level-3.md) | Use selection to compare numbers and test scores. |
| [Level 4](4.4-level-4.md) | Build a more challenging program that uses selection and arithmetic. |

---

## :material-clipboard-check-outline: Assessment Rubric

Each assessment level is worth **10 marks**.

For each level, you will be assessed against the performance bands below.

| Points | Level Title | Description |
|-----------|-----------|-----------|
| 1 | Beginning | Shows limited evidence of the skills for this level. |
| 4 | Developing | Shows some evidence of the skills for this level. |
| 7 | Progressing | Shows clear evidence of most skills for this level. |
| 10 | Secure | Shows clear and accurate evidence of all skills for this level. |

| Criterion | Description | Maximum Marks |
|-----------|-----------|-----------|
| Level 1 | Reading, correcting and explaining simple Python code. | 10 |
| Level 2 | Create short programs with inputs, variables, outputs and simple selection. | 10 |
| Level 3 | Create programs that use number input, conversion and selection to compare values. | 10 |
| Level 4 | Create programs that combine inputs, variables, conversion, arithmetic, selection and testing. | 10 |

**Total: 40 marks**

---

## :material-check-circle-outline: What to Submit

For each program task, submit evidence of your code.

Your teacher may ask you to:

- Paste your code into your work.
- Add a screenshot of your code.
- Add a screenshot of your program running.
- Explain what your code does.

!!! tip "Screenshot Reminder"

    Make sure screenshots are cropped and large enough to read clearly.

---

## :material-lightbulb-on: Assessment Tips

- Read each task carefully before writing code.
- Use clear variable names.
- Convert number inputs using `int()` or `float()`.
- Remember that `=` stores a value and `==` compares a value.
- Test your program with more than one input.
- Fix errors one at a time.
